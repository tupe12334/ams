# Enforce pedantic Clippy linting rules with CI check on every push

Configure the most pedantic Clippy linting rules and enforce them on every push via CI.

## Requirements

### Clippy Configuration

* Enable `#![warn(clippy::all)]` - all default lints
* Enable `#![warn(clippy::pedantic)]` - pedantic lints
* Enable `#![warn(clippy::nursery)]` - nursery lints (experimental but useful)
* Enable `#![warn(clippy::cargo)]` - cargo manifest lints
* Enable `#![deny(clippy::unwrap_used)]` - prevent unwrap in production code
* Enable `#![deny(clippy::expect_used)]` - prevent expect in production code
* Enable `#![deny(clippy::panic)]` - prevent panic in production code
* Enable `#![warn(missing_docs)]` - require documentation
* Enable `#![warn(rust_2018_idioms)]` - Rust 2018 idioms

### CI Integration

* Add GitHub Actions workflow for Clippy checks
* Run `cargo clippy -- -D warnings` on every push and PR
* Block merges if any lint warnings exist
* Run `cargo fmt --check` alongside Clippy

### Configuration File

* Add `clippy.toml` for project-specific Clippy configuration
* Configure allowed lint exceptions with justification comments

## Acceptance Criteria

* All pedantic Clippy lints pass
* CI fails on any lint warning
* No `#[allow(...)]` without documented justification
* `cargo fmt` check passes
* New code must pass all lints before merge
